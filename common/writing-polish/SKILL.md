# Writing Polish / 写作润色

根据用户输入的原始语句，用更 native、更学术化、更地道的中文表达进行润色改写，保持原意不变，仅优化表达方式。

## ⚠️ 核心规则

- **只输出润色后的语句**（或附带简要修改说明），**不将润色内容写入知识库**
- **不触发 git commit/push**（润色操作不产生知识库变更）
- 如果用户输入过于简短或缺少上下文，主动追问后再润色
- 保留原文中的标准英文术语（如 throughput、latency、baseline 等）

## 学术中文短语库

以下短语按功能分类，润色时参考替换生硬/口语化表达。

### 1. 转折 / 让步

| 生硬表达 | 推荐替换 |
|---------|---------|
| 但是 / 但 | 然而、不过、尽管如此、即便如此 |
| 虽然...但是 | 尽管...然而...、虽...但... |
| 相反 | 与之相反、恰恰相反、反观 |
| 不是...而是 | 并非...而是...、与其说...不如说... |
| 即使 | 即便、纵然 |

**例句**：
- 原文：这个方法很快，但是不够准确。
- 润色：该方法虽具备较高的执行效率，**然而**在准确度方面尚存不足。

### 2. 因果 / 顺承

| 生硬表达 | 推荐替换 |
|---------|---------|
| 所以 | 因此、故而、因而 |
| 因为...所以 | 由于...因此...、鉴于...故... |
| 然后 | 进而、随后、在此基础上 |
| 从而 | 由此、藉此 |
| 最后 | 最终、终究 |

**例句**：
- 原文：因为内存不够，所以系统变慢了。
- 润色：**鉴于**内存容量不足，系统性能**因此**出现显著劣化。

### 3. 递进 / 并列

| 生硬表达 | 推荐替换 |
|---------|---------|
| 而且 | 此外、不仅如此、更何况 |
| 还有 | 除此之外、另外、再者 |
| 不仅...而且 | 不仅...还...、不只...更... |
| 一方面...另一方面 | 一则...二则...、既...又... |

**例句**：
- 原文：这个系统不仅快，而且省内存。
- 润色：该系统**不仅**具备高吞吐性能，**更**在内存效率方面表现突出。

### 4. 程度 / 强调

| 程度 | 推荐短语 |
|------|---------|
| **极其重要** | 至关重要、不可或缺、举足轻重、具有决定性意义 |
| **非常严重** | 尤为突出、不可忽视、不容小觑、令人堪忧 |
| **很大** | 显著、大幅度、急剧、可观的 |
| **十分明显** | 显而易见、不言而喻、昭然若揭 |
| **特别关注** | 尤应重视、值得警惕、亟需关注 |
| **充分说明** | 足以表明、充分印证、有力佐证 |
| **深刻影响** | 深远影响、根本性改变、颠覆性冲击 |
| **普遍存在** | 屡见不鲜、习以为常、比比皆是 |

**例句**：
- 原文：这个问题很重要，不能不管。
- 润色：该问题**至关重要**，**不容忽视**。

### 5. 引出观点 / 论述

| 功能 | 推荐短语 |
|------|---------|
| 引入话题 | 就...而言、在...方面、关于...、就...来看 |
| 提出观点 | 本文认为、笔者认为、我们主张、可以认为 |
| 引用他人 | ...指出、...强调、...认为、据...的研究 |
| 举例说明 | 例如、以...为例、典型的例子是、不妨考虑 |
| 换言解释 | 换言之、也就是说、即、通俗而言 |
| 总结归纳 | 综上、总而言之、概括而言、归根结底 |
| 呼应前文 | 如前所述、基于以上分析、承接上文 |

**例句**：
- 原文：有人说这个方法好，但我觉得不一定。
- 润色：**有研究指出**该方法具有一定优势，**然而笔者**认为其适用性尚待验证。

### 6. 对比 / 比较

| 功能 | 推荐短语 |
|------|---------|
| 相似比较 | 与...类似、与...异曲同工、无独有偶 |
| 差异对比 | 相比于、相较于、与...不同之处在于 |
| 优劣对比 | 优于/胜于、逊于/不敌、各有千秋 |
| 变化趋势 | 由...转向...、从...演变为...、...逐渐被...取代 |

**例句**：
- 原文：跟旧方法比，新方法快很多。
- 润色：**相较于**传统方案，新方法在速度上**显著优于**前者。

### 7. 评价 / 态度

| 态度 | 推荐短语 |
|------|---------|
| 正面肯定 | 值得肯定、令人印象深刻、成效显著、可圈可点 |
| 谨慎乐观 | 有一定参考价值、尚需验证、初步表明、值得关注 |
| 批判质疑 | 有待商榷、值得推敲、存在争议、不尽如人意 |
| 中立客观 | 尚难定论、各有优劣、需辩证看待 |
| 意外发现 | 值得注意的是、有趣的是、令人惊讶的是 |

### 8. 逻辑连接 / 篇章结构

| 功能 | 推荐短语 |
|------|---------|
| 开启论述 | 首先、本文首先讨论... |
| 延续论述 | 其次、进一步地、在此基础上 |
| 深入分析 | 深入来看、更进一步而言、具体而言 |
| 补充说明 | 需要补充的是、值得一提的还有 |
| 回到主题 | 回到本文关注的核心问题、言归正传 |
| 结束论述 | 最后、综上所述、概括而言 |

### 9. 常见生硬表达的替换

| 生硬 / 口语 | 推荐替换 |
|------------|---------|
| 很多 | 大量、众多、诸多、广泛的 |
| 很好 | 出色、优异、卓越的、令人满意的 |
| 不好 | 欠佳、不尽如人意、有待改进 |
| 差不多 | 大致相当、基本持平、不相上下 |
| 越来越 | 日益、日趋、逐渐、与日俱增 |
| 真的 | 确实、的确、实实在在 |
| 基本上 | 大体上、总体而言、本质上 |
| 需要 | 亟需、有必要、应当 |
| 应该 | 应当、理应、宜 |
| 可能 | 或许、或可、有可能 |
| 显然 | 显而易见、不言而喻 |
| 主要是 | 主要归因于、核心原因在于 |
| 做出了贡献 | 贡献了、推动了、促进了 |
| 起了作用 | 发挥了作用、产生了影响 |

### 10. 技术论文常用句式模板

**问题陈述**：
- `<问题>是<领域>中长期存在/亟待解决/日益突出的核心挑战`
- `<现有方案>在<场景>下暴露了<问题>，制约了<目标>`

**方案介绍**：
- `本文提出<方案名>，通过<核心思路>，旨在解决<问题>`
- `<方案>的核心洞察在于：<一句话洞察>`

**贡献声明**：
- `本文的主要贡献包括：(1)...(2)...(3)...`
- `<方案>在<指标>上较<基线>提升了<数字>，同时<附带收益>`

**评估结论**：
- `实验结果表明，<方案>在<条件>下显著优于<对比对象>`
- `<结果>充分验证了<假设/设计>的有效性`

## 润色流程

1. **接收用户输入** → 识别需要润色的语句
2. **判断语境**（学术论文 / 技术博客 / 邮件沟通 / 日常表达）
3. **对照短语库** → 找出可替换的生硬表达
4. **输出润色版本** + 简要说明改了什么（可选）
5. **不触发知识库写入**（不更新 KNOWLEDGE.md, 不 commit, 不修改 reading-log.md）

## 示例

**用户输入**：润色一下："这个方法很快，而且很准，但是用的内存太多了，所以不太好。"

**润色输出**：
> 该方法虽然具备较高的执行效率和准确度，**然而**内存占用**过大**，**因此**整体表现**不尽如人意**。
>
> **修改说明**：将"很快"→"较高的执行效率"、"很准"→"较高的准确度"、"内存太多"→"内存占用过大"、"不太好"→"不尽如人意"，使表达更学术化。

## 注意事项

- 不要过度润色——如果原文已经很好，只做微调
- 不要改变原意——润色是表达优化，不是内容改写
- 不要删除技术细节——保留所有技术术语和数据
- 保持句子长度合理——不要为了"高级"把简单句改成复杂长句
